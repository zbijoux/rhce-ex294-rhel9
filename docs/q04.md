# Q04 — Use the SELinux role

## Question

> Create a playbook called `/home/student/ansible/playbooks/selinux.yml` that:
>
> - Runs on **all** managed nodes
> - Uses the `rhel-system-roles.selinux` role
> - Configures SELinux policy as `targeted`
> - Sets SELinux state as `enforcing`

---

## Approach

`rhel-system-roles` is installed as an RPM on the control node. The problem is that `ansible-navigator` runs inside an Execution Environment (EE) container — it cannot see RPM-installed roles in `/usr/share/ansible/roles` on the host. You have **two options** to make the role available to `ansible-navigator`:

**Option 1 — Copy the RPM role into your working roles directory**
Install the RPM on the control node, then copy the selinux role into `/home/student/ansible/roles/` so the EE picks it up via the mounted workspace.

**Option 2 — Install via the redhat.rhel_system_roles collection**
The `redhat-rhel_system_roles` collection was installed in Q05 into `./collections/`. Reference the role using its full collection FQCN `redhat.rhel_system_roles.selinux`. Both options allow `ansible-navigator run -m stdout`.

The playbook itself uses a `block/rescue` pattern because applying SELinux changes often requires a reboot — the rescue catches `selinux_reboot_required`, reboots the node, waits for it to return, then re-applies the role.

---

## Solution

### Option 1 — Copy RPM role into roles directory

```bash
# Step 1: Install the RPM on the control node
sudo dnf -y install rhel-system-roles

# Step 2: Copy the role into your working roles directory
cp -r /usr/share/ansible/roles/rhel-system-roles.selinux \
      /home/student/ansible/roles/

# Verify it is there
ansible-galaxy list
```

File: `playbooks/selinux.yml`

```yaml
---
- hosts: all
  become: true
  become_method: sudo
  become_user: root
  vars:
    selinux_policy: targeted
    selinux_state: enforcing
  tasks:
    - name: Execute SELinux role and catch reboot requirement
      block:
        - name: Include selinux role
          ansible.builtin.include_role:
            name: rhel-system-roles.selinux
      rescue:
        - name: Fail if error is not reboot-related
          ansible.builtin.fail:
            msg: "role failed"
          when: not selinux_reboot_required

        - name: Reboot managed host
          ansible.builtin.reboot:

        - name: Wait for managed host to come back
          ansible.builtin.wait_for_connection:
            delay: 10
            timeout: 300

        - name: Re-apply the selinux role after reboot
          ansible.builtin.include_role:
            name: rhel-system-roles.selinux
```

---

### Option 2 — Collection FQCN (redhat.rhel_system_roles)

The `redhat-rhel_system_roles` collection installed in Q05 already includes the selinux role. Reference it with its full FQCN — no copy step needed.

File: `playbooks/selinux.yml`

```yaml
---
- hosts: all
  become: true
  become_method: sudo
  become_user: root
  vars:
    selinux_policy: targeted
    selinux_state: enforcing
  tasks:
    - name: Execute SELinux role and catch reboot requirement
      block:
        - name: Include selinux role via collection FQCN
          ansible.builtin.include_role:
            name: redhat.rhel_system_roles.selinux
      rescue:
        - name: Fail if error is not reboot-related
          ansible.builtin.fail:
            msg: "role failed"
          when: not selinux_reboot_required

        - name: Reboot managed host
          ansible.builtin.reboot:

        - name: Wait for managed host to come back
          ansible.builtin.wait_for_connection:
            delay: 10
            timeout: 300

        - name: Re-apply the selinux role after reboot
          ansible.builtin.include_role:
            name: redhat.rhel_system_roles.selinux
```

---

## How to run

Both options use `ansible-navigator`:

```bash
cd /home/student/ansible

# Option 1 only — copy the role first
sudo dnf -y install rhel-system-roles
cp -r /usr/share/ansible/roles/rhel-system-roles.selinux roles/

# Both options — run with ansible-navigator
ansible-navigator run playbooks/selinux.yml -m stdout
```

> `ansible-playbook` also works for Option 1 if the role is reachable via `roles_path`.

---

## Verification

```bash
# Check SELinux config file on all nodes
ansible all -m shell -a 'grep ^SELINUX= /etc/selinux/config'
# Expected: SELINUX=enforcing

# Check runtime enforcement status
ansible all -m shell -a 'getenforce'
# Expected: Enforcing on all 5 nodes
```

---

## Common exam traps

- **Option 1: copy, don't just install** — `ansible-navigator` runs inside a container EE and cannot see `/usr/share/ansible/roles` on the host. You must copy the role into your working `roles/` directory.
- **Option 2: full FQCN required** — use `redhat.rhel_system_roles.selinux`, not the short name `selinux`. The short name resolves to nothing inside the EE.
- **The rescue block must check `selinux_reboot_required`** — without the `when: not selinux_reboot_required` guard, any non-reboot failure silently triggers a reboot instead of failing visibly.
- **`selinux_policy: targeted`** — must be specified explicitly; do not leave it out or assume it defaults to targeted.
- **`ansible-navigator run -m stdout`** — the `-m stdout` flag gives you clean terminal output instead of the TUI interface, which is much easier to read during the exam.
