# Q17 — Configure a cron job

## Question

> Create a playbook called `/home/student/ansible/playbooks/cron.yml` that runs on hosts in the **`dev`** host group and creates a cron job for user `natasha` as follows:
>
> - Runs every 2 minutes
> - Executes: `logger "EX294 in progress"`

---

## Approach

Use `ansible.builtin.cron`. Set `minute: "*/2"` for every-2-minute frequency. Set `user: natasha` so the cron entry is written to natasha's crontab. Give the job a `name:` for idempotency.

---

## Solution

File: `playbooks/cron.yml`

```yaml
---
- name: Configure cron job for natasha on dev hosts
  hosts: dev
  tasks:
    - name: Create cron job that logs EX294 progress every 2 minutes
      ansible.builtin.cron:
        name: "EX294 progress logger"
        minute: "*/2"
        job: 'logger "EX294 in progress"'
        user: natasha
```

---

## How to run

```bash
cd /home/student/ansible
ansible-navigator run playbooks/cron.yml -m stdout
```

---

## Verification

```bash
ansible dev -a 'crontab -l -u natasha'
# Expected:
# #Ansible: EX294 progress logger
# */2 * * * * logger "EX294 in progress"
```

## Common exam traps

- **`hosts: dev` not `hosts: test`** — re-read the question. The cron job is for dev nodes only. The PDF example accidentally targets test.
- **`minute: "*/2"` not `minute: "2"`** — `"2"` means "at minute 2 of every hour" (once per hour). `"*/2"` means "every 2 minutes".
- **`job` string must be exact** — `logger "EX294 in progress"` with the quotes around the message. Missing quotes causes logger to misparse the arguments.
- **PDF typo** — the PDF example uses `EX200 in progress`. The correct string for this exam is `EX294 in progress`.
- **`user: natasha`** — without this, the cron job is created for the `student` (or `root`) user, not natasha.
