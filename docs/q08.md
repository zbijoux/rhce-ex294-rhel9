# Q08 — Use roles from Ansible Galaxy

## Question

> Create a playbook called `/home/student/ansible/playbooks/roles.yml` with the following requirements:
>
> - A play that runs on hosts in the `balancers` host group and uses the `balancer` role
>   - Browsing to the balancer IP alternates between `node3` and `node4` output
> - A play that runs on hosts in the `webservers` host group and uses the `phpinfo` role
>   - Browsing to `/hello.php` on webservers shows `Hello PHP World from FQDN`

---

## Approach

Write two plays in one playbook. The `phpinfo` play must come before the `balancer` play so the backend web nodes are ready before the load balancer is configured. Both roles were installed in Q06.

---

## Solution

File: `playbooks/roles.yml`

```yaml
---
- name: Deploy phpinfo role on webservers
  hosts: webservers
  roles:
    - phpinfo

- name: Deploy balancer role on balancers
  hosts: balancers
  roles:
    - balancer
```

---

## How to run

> **Important:** Use `ansible-navigator` (not `ansible-playbook`) for Galaxy-installed collection roles.

```bash
cd /home/student/ansible

# Ensure roles are installed first (Q06)
ansible-galaxy install -r roles/requirements.yml

ansible-navigator run playbooks/roles.yml -m stdout
```

---

## Verification

```bash
# Test load balancer round-robin
curl http://172.25.250.13
# First request: Welcome to node3.lab.example.com on 172.25.250.11

curl http://172.25.250.13
# Second request: Welcome to node4.lab.example.com on 172.25.250.12

# Test phpinfo pages
curl http://node3/hello.php
# Expected: Hello PHP World from node3.lab.example.com

curl http://node4/hello.php
# Expected: Hello PHP World from node4.lab.example.com
```

---

## Common exam traps

- **Play order matters** — `phpinfo` play must run before `balancer` play. The balancer configures HAProxy backends; if the backends aren't ready first, curl to the balancer may return errors.
- **Use `ansible-navigator` not `ansible-playbook`** — Galaxy-downloaded collection roles require the execution environment.
- **Curl the balancer IP multiple times** — one successful curl is not enough. Verify round-robin alternates between node3 and node4.
- **Ensure Q06 roles are installed** — if `balancer` or `phpinfo` are missing, this playbook fails immediately.
