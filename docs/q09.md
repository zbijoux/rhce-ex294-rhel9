# Q09 — Create and use a logical volume

## Question

> Create a playbook called `/home/student/ansible/playbooks/lv.yml` that runs on all managed nodes and:
>
> - Creates a logical volume with these requirements:
>   - Created in the `research` volume group
>   - Logical volume name: `data`
>   - Logical volume size: `1500 MiB`
>   - Formatted with the `ext4` filesystem
> - If the requested size cannot be created, displays the error message `Could not create logical volume of that size` and uses `800 MiB` instead
> - If the volume group `research` does not exist, displays `Volume group does not exist`
> - Does **NOT** mount the logical volume

---

## Approach

Use `block/rescue` to handle the size fallback. Wrap the whole block in a `when:` condition that checks `ansible_lvm.vgs.research is defined`. Use `community.general.lvol` and `community.general.filesystem`. The `when:` must be on the `block`, not on individual tasks.

---

## Solution

File: `playbooks/lv.yml`

```yaml
---
- name: Create logical volume on all managed nodes
  hosts: all
  tasks:
    - name: Create LV in research VG (1500 MiB, fallback 800 MiB)
      block:
        - name: Create logical volume data at 1500 MiB
          community.general.lvol:
            vg: research
            lv: data
            size: 1500

        - name: Format logical volume with ext4
          community.general.filesystem:
            fstype: ext4
            dev: /dev/research/data

      rescue:
        - name: Notify size failure
          ansible.builtin.debug:
            msg: "Could not create logical volume of that size"

        - name: Create logical volume data at 800 MiB (fallback)
          community.general.lvol:
            vg: research
            lv: data
            size: 800

        - name: Format fallback logical volume with ext4
          community.general.filesystem:
            fstype: ext4
            dev: /dev/research/data

      when: ansible_lvm.vgs.research is defined

    - name: Notify missing volume group
      ansible.builtin.debug:
        msg: "Volume group does not exist"
      when: ansible_lvm.vgs.research is not defined
```

---

## How to run

```bash
cd /home/student/ansible
ansible-navigator run playbooks/lv.yml -m stdout
```

Watch the PLAY RECAP carefully:
- `rescued=1` on node3 means it fell back to 800 MiB (correct)
- `skipped=2` on node1 means the VG didn't exist (correct)

---

## Verification

```bash
# Check LV sizes on all nodes
ansible all -m shell -a 'lvs'
# node2, node4, node5: data  research  1.47g
# node3:               data  research  800.00m
# node1: (no output — no VG)

# Verify ext4 filesystem
ansible all -m shell -a 'blkid /dev/research/data'
# Expected: TYPE="ext4" on nodes 2,3,4,5
# node1: non-zero return code (expected)
```

## Common exam traps

- **`when:` on the `block`, not on individual tasks** — if you put `when:` on each task, the rescue block also inherits the condition and the fallback 800 MiB tasks also get skipped when the VG is missing.
- **Do NOT mount the LV** — the question explicitly forbids it. No `mount` module, no `/etc/fstab` entry.
- **`community.general` FQCN required** — plain `lvol:` may work if the collection is in the path, but FQCN is safer and exam-correct.
- **`rescued=1` in the recap is expected** — it does not mean the playbook failed. Check that the LV was created at 800 MiB.
