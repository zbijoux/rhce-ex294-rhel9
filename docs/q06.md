# Q06 — Install roles using Ansible Galaxy

## Question

> Use Ansible Galaxy with a requirements file called `/home/student/ansible/roles/requirements.yml` to download and install roles to the default `/home/student/ansible/roles` directory from the following URLs:
>
> - `http://classroom/materials/haproxy.tar` — the name of this role should be `balancer`
> - `http://classroom/materials/phpinfo.tar` — the name of this role should be `phpinfo`

---

## Approach

Create `roles/requirements.yml` as a YAML list (not a dict with a `roles:` key). Each entry has `src:` (the URL) and `name:` (the desired role name after install). Run `ansible-galaxy install -r`.

---

## Solution

File: `roles/requirements.yml`

```yaml
---
- src: http://classroom/materials/haproxy.tar
  name: balancer

- src: http://classroom/materials/phpinfo.tar
  name: phpinfo
```

### Install command

```bash
cd /home/student/ansible
ansible-galaxy install -r roles/requirements.yml
```

---

## How to run

This question has no playbook. The install command above is the solution.

```bash
# Verify both roles appear in the roles list
ansible-galaxy list
```

---

## Verification

```bash
ansible-galaxy list
# Expected:
# - balancer, ...
# - phpinfo, ...

ls /home/student/ansible/roles/
# Expected: apache/  balancer/  phpinfo/
```

---

## Common exam traps

- **`name:` renames the role** — without `name: balancer`, the role installs as `haproxy`. The exam checks for `balancer` specifically.
- **Roles requirements use a root-level list, not a `roles:` key** — unlike the collections requirements format which uses `collections:`.
- **`ansible-galaxy install` not `ansible-galaxy collection install`** — wrong subcommand installs nothing and exits silently.
- **Default path** — roles install to the first path in `roles_path`. Confirm `ansible.cfg` has `/home/student/ansible/roles` as the first entry.
