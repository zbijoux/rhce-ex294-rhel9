# Q05 — Install a collection

## Question

> Install the following collection artifacts available from `http://classroom/materials/` to the control node.
> The collections should be installed into the default collections directory `/home/student/ansible/collections`:
>
> - `redhat-insights-1.0.7.tar.gz`
> - `community-general-5.5.0.tar.gz`
> - `redhat-rhel_system_roles-1.19.3.tar.gz`

---

## Approach

Create a `collections/requirements.yml` file with a `collections:` key listing the three tarball URLs. Run `ansible-galaxy collection install` with `-r` and `-p` flags. Verify with `ansible-navigator collections`.

---

## Solution

### Step 1 — requirements file

File: `collections/requirements.yml`

```yaml
---
collections:
  - name: http://classroom/materials/redhat-insights-1.0.7.tar.gz
  - name: http://classroom/materials/community-general-5.5.0.tar.gz
  - name: http://classroom/materials/redhat-rhel_system_roles-1.19.3.tar.gz
```

### Step 2 — Install

```bash
cd /home/student/ansible
ansible-galaxy collection install \
  -r collections/requirements.yml \
  -p /home/student/ansible/collections
```

---

## How to run

This question has no playbook. The installation command above is the solution.

```bash
# Verify collections are available in ansible-navigator
ansible-navigator collections

# Test a specific collection module is accessible
ansible-navigator doc community.general.filesystem -m stdout
```

---

## Verification

```bash
ansible-navigator collections
# Expected: redhat.insights, community.general, redhat.rhel_system_roles listed

ls /home/student/ansible/collections/ansible_collections/
# Expected: community/  redhat/
```

---

## Common exam traps

- **Use `-p` not `--collections-path`** — the `-p` short flag reliably targets the correct directory; the long form can behave differently depending on environment.
- **`collections:` key at root level** — do not nest it under `roles:`; they are separate keys in the same file or separate files.
- **`collections_path` in `ansible.cfg` must list `./collections/` first** — otherwise `ansible-navigator` finds the system path before your installed collections.
- **Do not confuse this with Q06** — Q05 installs collections; Q06 installs roles. Different commands, different directories.
