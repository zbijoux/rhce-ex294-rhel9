# Q10 — Generate a hosts file

## Question

> Download an initial template file from `http://classroom/materials/hosts.j2` to `/home/student/ansible/templates/`.
> Complete the template so that it generates a file with a line for each inventory host.
>
> Download the playbook from `http://classroom/materials/hosts.yml` to `/home/student/ansible/playbooks/`.
> **Do not make any changes to the playbook.**
>
> When the playbook runs, `/etc/myhosts` on hosts in the `dev` host group should contain:
>
> ```
> 127.0.0.1 localhost localhost.localdomain localhost4 localhost4.localdomain4
> ::1 localhost localhost.localdomain localhost6 localhost6.localdomain6
> 172.25.250.9  node1.lab.example.com node1
> 172.25.250.10 node2.lab.example.com node2
> 172.25.250.11 node3.lab.example.com node3
> 172.25.250.12 node4.lab.example.com node4
> 172.25.250.13 node5.lab.example.com node5
> ```

---

## Approach

Download both files. Edit only the template. Use a Jinja2 `for` loop over `groups.all` and pull the IP and FQDN from `hostvars` using the `ansible_facts` namespace — not the shorthand magic variables.

---

## Solution

### Step 1 — Download files

```bash
cd /home/student/ansible
wget -O templates/hosts.j2 http://classroom/materials/hosts.j2
wget -O playbooks/hosts.yml http://classroom/materials/hosts.yml
```

### Step 2 — Complete the template

File: `templates/hosts.j2`

```jinja
127.0.0.1 localhost localhost.localdomain localhost4 localhost4.localdomain4
::1 localhost localhost.localdomain localhost6 localhost6.localdomain6
{% for host in groups.all %}
{{ hostvars[host].ansible_facts.default_ipv4.address }} {{ hostvars[host].ansible_facts.fqdn }} {{ host }}
{% endfor %}
```

---

## How to run

> Do NOT modify `playbooks/hosts.yml`.

```bash
cd /home/student/ansible
ansible-navigator run playbooks/hosts.yml -m stdout
```

---

## Verification

```bash
ansible dev -m shell -a 'cat /etc/myhosts'
# Expected: all 5 nodes listed with correct IPs and FQDNs
```

---

## Common exam traps

- **`hostvars[host].ansible_facts.fqdn` not `hostvars[host].ansible_fqdn`** — when facts are gathered through `ansible_facts` namespace, the shorthand `ansible_fqdn` may not be available in `hostvars` for other hosts.
- **`groups.all` not `groups['dev']`** — the template must include ALL inventory hosts, not just dev nodes.
- **Do NOT edit `hosts.yml`** — the exam instruction is explicit. Only `hosts.j2` is yours to change.
- **Fact gathering must run on all hosts** — the playbook must `gather_facts: yes` (or not disable it) so `hostvars` is populated for all nodes before the template runs on dev.
