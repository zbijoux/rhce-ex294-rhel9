# Q16 — Rekey an Ansible vault

## Question

> Rekey an existing Ansible vault as follows:
>
> - Download the vault from `http://classroom/materials/salaries.yml` to `/home/student/ansible/`
> - The current vault password is `insecure4sure`
> - The new vault password is `bbe2de98389b`
> - The vault must remain in an encrypted state with the new password

---

## Approach

Download the file, then use `ansible-vault rekey --ask-vault-pass`. You will be prompted for the old password first, then the new password twice. Verify the file is still encrypted and readable with the new password.

---

## Solution

```bash
# Download the vault
wget -O /home/student/ansible/salaries.yml http://classroom/materials/salaries.yml

# Rekey the vault
ansible-vault rekey --ask-vault-pass /home/student/ansible/salaries.yml
# Vault password: insecure4sure
# New Vault password: bbe2de98389b
# Confirm New Vault password: bbe2de98389b
# Output: Rekey successful
```

---

## How to run

This question has no playbook. The rekey command above is the complete solution.

---

## Verification

```bash
# Confirm file is still encrypted
head -1 /home/student/ansible/salaries.yml
# Expected: $ANSIBLE_VAULT;1.1;AES256

# View contents with new password
ansible-vault view --ask-vault-pass /home/student/ansible/salaries.yml
# New Vault password: bbe2de98389b
# (contents should be visible)
```

---

## Common exam traps

- **Exact passwords** — type carefully. `insecure4sure` (old) and `bbe2de98389b` (new). The PDF has a typo in the example; use these exact strings.
- **File must remain encrypted** — do NOT run `ansible-vault decrypt`. After rekeying, `head -1` must still show `$ANSIBLE_VAULT`.
- **`vault_password_file` in `ansible.cfg` affects `--ask-vault-pass`** — if `secret.txt` is configured, `rekey` will use it for the OLD password automatically. You'll only be prompted for the new password. This is fine — just enter the new password when asked.
- **Use `--ask-vault-pass` not `--vault-password-file`** — you're changing the password, so you must provide both old and new interactively.
