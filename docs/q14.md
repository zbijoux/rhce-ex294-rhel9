# Q14 — Create a password vault

## Question

> Create an Ansible vault to store user passwords as follows:
>
> - The name of the vault is `/home/student/ansible/locker.yml`
> - The vault contains two variables:
>   - `pw_developer` with value `Imadev`
>   - `pw_manager` with value `Imamgr`
> - The password to encrypt and decrypt the vault is `whenyouwishuponastar`
> - The password is stored in the file `/home/student/ansible/secret.txt`

---

## Approach

Write the vault password to `secret.txt` first. Add `vault_password_file` to `ansible.cfg` so all subsequent vault operations use it automatically. Then create the vault with `ansible-vault create`.

---

## Solution

### Step 1 — Create the password file

```bash
echo "whenyouwishuponastar" > /home/student/ansible/secret.txt
chmod 600 /home/student/ansible/secret.txt
```

### Step 2 — Reference it in ansible.cfg

In `ansible.cfg` under `[defaults]`:

```ini
vault_password_file = /home/student/ansible/secret.txt
```

### Step 3 — Create the vault

```bash
cd /home/student/ansible
ansible-vault create locker.yml
```

Enter the vault contents:

```yaml
---
pw_developer: Imadev
pw_manager: Imamgr
```

---

## How to run

This question has no playbook. The vault is used in Q15.

```bash
# Verify the vault is encrypted
cat locker.yml
# Expected: $ANSIBLE_VAULT;1.1;AES256 ...

# View the vault contents (uses secret.txt automatically via ansible.cfg)
ansible-vault view locker.yml
```

## Common exam traps

- **`vault_password_file` path must be absolute** — `~/ansible/secret.txt` may not resolve correctly. Use the full path.
- **`secret.txt` must not have a trailing newline from a text editor** — `echo` adds one newline which is fine; most tools handle it. But `echo -n` is safer if unsure.
- **Do not use `ansible-vault encrypt`** — that encrypts an existing plaintext file. Use `ansible-vault create` to write the encrypted file directly.
- **`locker.yml` must remain encrypted** — never run `ansible-vault decrypt` on it. The exam grader checks the file is encrypted.
