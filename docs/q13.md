# Q13 — Generate a hardware report

## Question

> Create a playbook called `/home/student/ansible/playbooks/hwreport.yml` that produces an output file `/root/hwreport.txt` on all managed nodes with:
>
> - Inventory host name
> - Total memory in MB
> - BIOS version
> - Size of disk device `vda`
> - Size of disk device `vdb`
>
> Each line contains a single `key=value` pair.
> If a hardware item does not exist, the value should be `NONE`.

---

## Approach

Download the empty template with `get_url`. Use five `lineinfile` tasks to replace the placeholder lines using `regexp:` anchors. Use the `default('NONE', true)` filter for facts that may not exist on all nodes (e.g. `vdb` on node1).

---

## Solution

File: `playbooks/hwreport.yml`

```yaml
---
- name: Generate hardware report on all managed nodes
  hosts: all
  tasks:
    - name: Download empty hardware report template
      ansible.builtin.get_url:
        url: http://materials/hwreport.empty
        dest: /root/hwreport.txt
        force: yes

    - name: Set HOST value
      ansible.builtin.lineinfile:
        path: /root/hwreport.txt
        regexp: '^HOST='
        line: "HOST={{ inventory_hostname }}"

    - name: Set MEMORY value
      ansible.builtin.lineinfile:
        path: /root/hwreport.txt
        regexp: '^MEMORY='
        line: "MEMORY={{ ansible_memtotal_mb | default('NONE', true) }}"

    - name: Set BIOS version
      ansible.builtin.lineinfile:
        path: /root/hwreport.txt
        regexp: '^BIOS='
        line: "BIOS={{ ansible_bios_version | default('NONE', true) }}"

    - name: Set DISK_SIZE_VDA
      ansible.builtin.lineinfile:
        path: /root/hwreport.txt
        regexp: '^DISK_SIZE_VDA='
        line: "DISK_SIZE_VDA={{ ansible_devices.vda.size | default('NONE', true) }}"

    - name: Set DISK_SIZE_VDB
      ansible.builtin.lineinfile:
        path: /root/hwreport.txt
        regexp: '^DISK_SIZE_VDB='
        line: "DISK_SIZE_VDB={{ ansible_devices.vdb.size | default('NONE', true) }}"
```

---

## How to run

```bash
cd /home/student/ansible
ansible-navigator run playbooks/hwreport.yml -m stdout
```

## Verification

```bash
ansible all -a 'cat /root/hwreport.txt'
```

Expected on a node with both disks:
```
HOST=node2
MEMORY=960
BIOS=1.15.0-1.el9
DISK_SIZE_VDA=10.00 GB
DISK_SIZE_VDB=1.00 GB
```

Expected on node1 (no vdb):
```
HOST=node1
MEMORY=5668
BIOS=1.15.0-1.el9
DISK_SIZE_VDA=20.00 GB
DISK_SIZE_VDB=NONE
```

---

## Common exam traps

- **`default('NONE', true)` — the second argument `true` is critical** — without it, an empty string value does not fall back to `NONE`. The `true` flag forces the default even for empty/falsy values.
- **`force: yes` on `get_url`** — ensures the template is re-downloaded fresh on each run rather than skipping if the file already exists.
- **`regexp:` must match the key prefix** — `'^HOST='` anchors to the start of line. Without the caret, `lineinfile` might match mid-line text.
- **Do not use `template` module** — the question says to download the empty file and modify it in-place with `lineinfile`.
