# Q03 — Install packages

## Question

> Create a playbook called `/home/student/ansible/playbooks/packages.yml` that:
>
> - Installs the `php` and `mariadb` packages on hosts in the `dev`, `test`, and `prod` host groups
> - Installs the `RPM Development Tools` packages group on hosts in the `dev` host group
> - Updates all packages to the latest version on hosts in the `dev` host group

---

## Approach

Write two separate plays in one playbook:
- **Play 1** targets `dev,test,prod` — installs php and mariadb
- **Play 2** targets `dev` only — installs the RPM Development Tools group and runs a full upgrade

Use `name: "*"` with `state: latest` for the upgrade. Group installs require the `@GroupName` syntax.

---

## Solution

File: `playbooks/packages.yml`

```yaml
---
- name: Install php and mariadb on dev, test and prod
  hosts: dev,test,prod
  tasks:
    - name: Install required packages
      ansible.builtin.yum:
        name:
          - php
          - mariadb
        state: present

- name: Install RPM Development Tools and upgrade all packages on dev
  hosts: dev
  tasks:
    - name: Install RPM Development Tools group
      ansible.builtin.yum:
        name: "@RPM Development Tools"
        state: present

    - name: Upgrade all packages to latest version
      ansible.builtin.yum:
        name: "*"
        state: latest
```

---

## How to run

```bash
cd /home/student/ansible
ansible-navigator run playbooks/packages.yml -m stdout
```

---

## Verification

```bash
# Confirm php and mariadb installed on dev, test, prod
ansible dev,test,prod -a 'rpm -q php mariadb'

# Confirm RPM Development Tools group installed on dev only
ansible dev -a 'yum grouplist installed'

# Confirm all packages are at latest on dev
ansible dev -a 'yum check-update'
```

---

## Common exam traps

- **`"@RPM Development Tools"`** — the `@` prefix is required for group installs; without it yum tries to install a package literally named `RPM Development Tools`.
- **Two plays, not two task blocks** — the host targets are different (`dev,test,prod` vs `dev`), so they must be separate plays.
- **`name: "*"` with `state: latest`** — not `state: present`. Using `present` with `*` does nothing useful.
- **Order matters** — the upgrade task should come after the group install task in the same play.
