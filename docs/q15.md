# Q15 — Create user accounts

## Question

> Download a list of users from `http://classroom/materials/user_list.yml` and save it to `/home/student/ansible/`.
>
> Using the vault `/home/student/ansible/locker.yml` created in Q14, create a playbook called `/home/student/ansible/playbooks/users.yml` that:
>
> - Users with job `developer`:
>   - Created on `dev` and `test` host groups
>   - Password from `pw_developer` variable
>   - Password expires after 30 days
>   - Member of supplementary group `devops`
> - Users with job `manager`:
>   - Created on `prod` host group
>   - Password from `pw_manager` variable
>   - Password expires after 30 days
>   - Member of supplementary group `opsmgr`
> - Passwords use SHA512 hash format

---

## Approach

Download `user_list.yml`. Write two plays. Each play loads both `locker.yml` and `user_list.yml` via `vars_files`. Create the supplementary group first, then loop over the `users` variable with a `when:` filter on `item.job`.

---

## Solution

### Step 1 — Download user list

```bash
wget -O /home/student/ansible/user_list.yml http://classroom/materials/user_list.yml
cat /home/student/ansible/user_list.yml
```

### Step 2 — Playbook

File: `playbooks/users.yml`

```yaml
---
- name: Create developer users on dev and test hosts
  hosts: dev,test
  vars_files:
    - /home/student/ansible/locker.yml
    - /home/student/ansible/user_list.yml
  tasks:
    - name: Ensure devops group exists
      ansible.builtin.group:
        name: devops
        state: present

    - name: Create developer user accounts
      ansible.builtin.user:
        name: "{{ item.name }}"
        groups: devops
        password: "{{ pw_developer | password_hash('sha512') }}"
        password_expire_max: "{{ item.password_expire_max }}"
      loop: "{{ users }}"
      when: item.job == 'developer'

- name: Create manager users on prod hosts
  hosts: prod
  vars_files:
    - /home/student/ansible/locker.yml
    - /home/student/ansible/user_list.yml
  tasks:
    - name: Ensure opsmgr group exists
      ansible.builtin.group:
        name: opsmgr
        state: present

    - name: Create manager user accounts
      ansible.builtin.user:
        name: "{{ item.name }}"
        groups: opsmgr
        password: "{{ pw_manager | password_hash('sha512') }}"
        password_expire_max: "{{ item.password_expire_max }}"
      loop: "{{ users }}"
      when: item.job == 'manager'
```

---

## How to run

```bash
cd /home/student/ansible

# The vault password is read automatically from secret.txt via ansible.cfg
ansible-navigator run playbooks/users.yml -m stdout
```

---

## Verification

```bash
# Check developer users on dev/test
ansible dev,test -m shell -a 'id bob; id fred'
# Expected: uid=... groups=...,devops

# Check manager users on prod
ansible prod -m shell -a 'id sally'
# Expected: uid=... groups=...,opsmgr

# Test login with password
ssh bob@node1
# Password: Imadev

ssh sally@node3
# Password: Imamgr
```

---

## Common exam traps

- **`vars_files` paths must be absolute** — relative paths break when the playbook is run from a different working directory.
- **`password_expire_max` not `password_expire`** — check with `ansible-doc ansible.builtin.user`. The correct key is `password_expire_max`.
- **Group task before user task** — adding a user to a non-existent supplementary group fails. Always create the group first.
- **`password_hash('sha512')`** — the password must be hashed. Passing the plain text string as `password:` will set the literal string as the hash (invalid login).
- **`when: item.job == 'developer'`** — the loop iterates all users; the `when:` condition filters to only the right job type per play.
