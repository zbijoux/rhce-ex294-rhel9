# Q07 — Create and use the apache role

## Question

> Create a role called `apache` in `/home/student/ansible/roles` with the following requirements:
>
> - The `httpd` package is installed, enabled on boot, and started
> - The firewall is enabled and running with a rule to allow access to the web server
> - A template file `index.html.j2` exists and is used to create `/var/www/html/index.html` with output:
>   `Welcome to HOSTNAME on IPADDRESS`
>   where HOSTNAME is the FQDN and IPADDRESS is the IP of the managed node
>
> Create a playbook called `/home/student/ansible/playbooks/apache.yml` that:
>
> - Runs on hosts in the `webservers` host group
> - Uses the `apache` role

---

## Approach

Use `ansible-galaxy role init` to create the role skeleton. Populate `tasks/main.yml` with four tasks: install httpd, start/enable httpd, start/enable firewalld, open HTTP in firewall, deploy the template. The template uses `ansible_fqdn` and `ansible_default_ipv4.address` magic variables.

---

## Solution

### Step 1 — Initialise role skeleton

```bash
cd /home/student/ansible
ansible-galaxy role init --init-path roles apache
```

### Step 2 — tasks/main.yml

File: `roles/apache/tasks/main.yml`

```yaml
---
- name: Install Apache httpd package
  ansible.builtin.yum:
    name: httpd
    state: latest

- name: Start and enable httpd service
  ansible.builtin.systemd:
    name: httpd
    state: started
    enabled: yes

- name: Start and enable firewalld service
  ansible.builtin.systemd:
    name: firewalld
    state: started
    enabled: yes

- name: Allow HTTP through firewall (permanent + immediate)
  ansible.posix.firewalld:
    service: http
    permanent: yes
    state: enabled
    immediate: yes

- name: Deploy index.html from Jinja2 template
  ansible.builtin.template:
    src: index.html.j2
    dest: /var/www/html/index.html
```

### Step 3 — templates/index.html.j2

File: `roles/apache/templates/index.html.j2`

```
Welcome to {{ ansible_fqdn }} on {{ ansible_default_ipv4.address }}
```

### Step 4 — Playbook

File: `playbooks/apache.yml`

```yaml
---
- name: Deploy apache role on webservers group
  hosts: webservers
  roles:
    - apache
```

---

## How to run

```bash
cd /home/student/ansible
ansible-navigator run playbooks/apache.yml -m stdout
```

---

## Verification

```bash
# Check httpd is running on webservers (node3, node4)
ansible webservers -a 'systemctl status httpd'

# Check firewall allows http
ansible webservers -a 'firewall-cmd --list-services'

# Curl each webserver node directly
curl http://node3
# Expected: Welcome to node3.lab.example.com on 172.25.250.11

curl http://node4
# Expected: Welcome to node4.lab.example.com on 172.25.250.12
```

## Common exam traps

- **Template variables** — use `ansible_fqdn` and `ansible_default_ipv4.address`. Do not use `inventory_hostname` (short name only) or `ansible_hostname` (also short).
- **`firewalld` must be started BEFORE the firewalld module task** — if firewalld is not running, `ansible.posix.firewalld` fails with a D-Bus error.
- **`hosts: webservers` not `hosts: prod`** — the play must target the `webservers` group (the parent), not the `prod` child group directly.
- **`immediate: yes`** — without this, the firewall rule only takes effect after the next firewalld reload/restart.
- **`ansible.posix.firewalld` requires the `ansible.posix` collection** — ensure it is available or install it.
