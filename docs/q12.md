# Q12 — Create a web content directory

## Question

> Create a playbook called `/home/student/ansible/playbooks/webcontent.yml` that runs on managed nodes in the `dev` host group and:
>
> - Creates the directory `/webdev` with:
>   - Owned by the `webdev` group
>   - Permissions: owner=rwx, group=rwx, other=rx
>   - Special permission: set group ID (setgid)
> - Symbolically links `/var/www/html/webdev` to `/webdev`
> - Creates the file `/webdev/index.html` with the content `Development`
> - Browsing `http://172.25.250.9/webdev/` returns `Development`

---

## Approach

Three tasks: `file` to create the directory with mode `2775` (setgid + rwxrwxr-x), `file` to create the symlink, and `copy` to write `index.html` with the correct SELinux type so Apache can serve it.

---

## Solution

File: `playbooks/webcontent.yml`

```yaml
---
- name: Create web content directory on dev hosts
  hosts: dev
  tasks:
    - name: Create /webdev directory with correct ownership and permissions
      ansible.builtin.file:
        path: /webdev
        state: directory
        group: webdev
        mode: '2775'

    - name: Create symbolic link from /var/www/html/webdev to /webdev
      ansible.builtin.file:
        src: /webdev
        dest: /var/www/html/webdev
        state: link

    - name: Create /webdev/index.html with correct SELinux context
      ansible.builtin.copy:
        content: "Development\n"
        dest: /webdev/index.html
        setype: httpd_sys_content_t
```

---

## How to run

```bash
cd /home/student/ansible
ansible-navigator run playbooks/webcontent.yml -m stdout
```

---

## Verification

```bash
# Check directory permissions and group
ansible dev -a 'ls -ldZ /webdev'
# Expected: drwxrwsr-x. webdev group, httpd_sys_content_t context

# Check the symlink exists
ansible dev -a 'ls -la /var/www/html/webdev'

# Curl the dev node webdev path
curl http://172.25.250.9/webdev/
# Expected: Development
```

---

## Common exam traps

- **`mode: '2775'` must be a quoted string** — without quotes YAML may interpret `2775` as a decimal integer (2775 ≠ octal 2775). Always quote octal modes.
- **`setype: httpd_sys_content_t` is required** — with SELinux enforcing (from Q04), Apache cannot read files with the default `default_t` context. Missing this causes a 403.
- **`webdev` group must exist on the node** — if it doesn't exist, the `file` task fails. Add a `group:` task before the `file` task if needed.
- **Symlink direction** — `src:` is the target (`/webdev`), `dest:` is the link path (`/var/www/html/webdev`). It is easy to reverse these.
