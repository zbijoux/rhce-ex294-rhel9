# Q01 — Install and configure Ansible

## Question

> Install and configure Ansible on the control node `control.lab.example.com` as follows:
>
> - Install the required packages
> - Create a static inventory file called `/home/student/ansible/inventory` so that:
>   - `node1` is a member of the `dev` host group
>   - `node2` is a member of the `test` host group
>   - `node3` and `node4` are members of the `prod` host group
>   - `node5` is a member of the `balancers` host group
>   - The `prod` group is a member of the `webservers` host group
> - Create a configuration file called `/home/student/ansible/ansible.cfg` so that:
>   - The host inventory file is `/home/student/ansible/inventory`
>   - The default roles directory is `/home/student/ansible/roles`
>   - The default collections directory is `/home/student/ansible/collections`

---

## Approach

1. SSH into the control node and install the package bundle
2. Create the working directory and subdirectories
3. Generate a base `ansible.cfg` using `ansible-config init --disabled`, then trim it to only the required keys
4. Write the static inventory with the correct group names and the `[webservers:children]` parent group

---

## Solution

### Step 1 — Install packages

```bash
ssh student@control
sudo dnf -y install ansible-automation-platform-common.noarch ansible-navigator
```

### Step 2 — Create directory structure

```bash
mkdir -p /home/student/ansible/roles
mkdir -p /home/student/ansible/collections
cd /home/student/ansible
```

### Step 3 — ansible.cfg

File: `ansible.cfg`

```ini
[defaults]
inventory         = /home/student/ansible/inventory
remote_user       = student
host_key_checking = False
roles_path        = /home/student/ansible/roles:/usr/share/ansible/roles
collections_path  = ./collections/:.ansible/collections:/usr/share/ansible/collections
vault_password_file = /home/student/ansible/secret.txt

[privilege_escalation]
become      = True
become_method = sudo
become_user = root
```

### Step 4 — inventory/hosts

File: `inventory/hosts`

```ini
[dev]
node1

[test]
node2

[prod]
node3
node4

[balancers]
node5

[webservers:children]
prod
```

---

## How to run

This question has no playbook — it is pure configuration.

```bash
# Verify Ansible version and config is loaded
ansible --version

# Verify inventory groups are correct
ansible-inventory --graph

# Smoke test connectivity to all nodes
ansible all -m ping
```

---

## Verification

```bash
ansible --version | grep 'config file'
# Expected: config file = /home/student/ansible/ansible.cfg

ansible-inventory --graph
# Expected tree showing dev, test, prod, balancers, webservers groups

ansible all -m ping
# Expected: SUCCESS pong from all 5 nodes
```

---

## Common exam traps

- **`roles_path` must include BOTH paths** — `/home/student/ansible/roles:/usr/share/ansible/roles`. Missing the system path breaks `rhel-system-roles` in Q04.
- **`collections_path` order matters** — the exam grader checks the directive value; `./collections/` must come first.
- **`remote_user = student`** — must match the SSH user on managed nodes. The PDF example uses `greg`; this repo uses `student`.
- **`[webservers:children]` not `[webservers]`** — the `:children` suffix is mandatory when adding a group as a member of another group.
- **`vault_password_file`** — add it now so Q14/Q15 work automatically without extra flags.
