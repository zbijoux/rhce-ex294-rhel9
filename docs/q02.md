# Q02 — Create yum repositories

## Question

> Create the playbook `/home/student/ansible/playbooks/yum_repo.yml` which creates the following yum repositories on **each of the managed nodes**:
>
> **Repository 1:**
> - Name: `EX294_BASE`
> - Description: `EX294 base software`
> - Base URL: `http://content/rhel9.0/x86_64/dvd/BaseOS`
> - GPG signature checking: enabled
> - GPG key URL: `http://content/rhel9.0/x86_64/dvd/RPM-GPG-KEY-redhat-release`
> - Repository: enabled
>
> **Repository 2:**
> - Name: `EX294_STREAM`
> - Description: `EX294 stream software`
> - Base URL: `http://content/rhel9.0/x86_64/dvd/AppStream`
> - GPG signature checking: enabled
> - GPG key URL: `http://content/rhel9.0/x86_64/dvd/RPM-GPG-KEY-redhat-release`
> - Repository: enabled

---

## Approach

Use `ansible.builtin.yum_repository`. Both repos share the same GPG key URL. The `file:` parameter controls the `.repo` filename written to `/etc/yum.repos.d/` — set it to match the repo name for clarity.

---

## Solution

File: `playbooks/yum_repo.yml`

```yaml
---
- name: Configure YUM repositories on all managed nodes
  hosts: all
  tasks:
    - name: Configure EX294_BASE repository
      ansible.builtin.yum_repository:
        file: EX294_BASE
        name: EX294_BASE
        description: EX294 base software
        baseurl: http://content/rhel9.0/x86_64/dvd/BaseOS
        gpgcheck: yes
        gpgkey: http://content/rhel9.0/x86_64/dvd/RPM-GPG-KEY-redhat-release
        enabled: yes

    - name: Configure EX294_STREAM repository
      ansible.builtin.yum_repository:
        file: EX294_STREAM
        name: EX294_STREAM
        description: EX294 stream software
        baseurl: http://content/rhel9.0/x86_64/dvd/AppStream
        gpgcheck: yes
        gpgkey: http://content/rhel9.0/x86_64/dvd/RPM-GPG-KEY-redhat-release
        enabled: yes
```

---

## How to run

```bash
cd /home/student/ansible
ansible-navigator run playbooks/yum_repo.yml -m stdout
```

---

## Verification

```bash
# Confirm repo files exist on all nodes
ansible all -a 'yum repoinfo'

# Install a test package to confirm repos work
ansible all -a 'yum -y install ftp'
ansible all -a 'rpm -q ftp'
```

Expected output per node:
```
ftp-X.X-X.el9.x86_64
```

## Common exam traps

- **GPG key URL typo** — the PDF shows a line-wrap in the URL. The correct URL is `RPM-GPG-KEY-redhat-release` (one continuous string, no line break).
- **`gpgcheck: yes` not `gpgcheck: 1`** — both work but `yes` is the idiomatic YAML boolean for this module.
- **`file:` vs `name:`** — `file:` sets the `.repo` filename on disk; `name:` sets the `[section]` header inside it. They can differ but keeping them the same avoids confusion.
- **`hosts: all`** — repos must be on ALL managed nodes, not just a subset.
