# Q11 — Modify file content

## Question

> Create a playbook called `/home/student/ansible/playbooks/issue.yml` as follows:
>
> - The playbook runs on all inventory hosts
> - Replaces the contents of `/etc/issue` with a single line of text:
>   - On hosts in the `dev` group: `Development`
>   - On hosts in the `test` group: `Test`
>   - On hosts in the `prod` group: `Production`

---

## Approach

One play targeting `all`. Three `copy` tasks, each guarded by a `when:` condition checking `inventory_hostname in groups.X`. The `content:` parameter replaces the entire file with the given string.

---

## Solution

File: `playbooks/issue.yml`

```yaml
---
- name: Set /etc/issue content based on host group
  hosts: all
  tasks:
    - name: Set /etc/issue to Development on dev hosts
      ansible.builtin.copy:
        content: "Development"
        dest: /etc/issue
      when: inventory_hostname in groups.dev

    - name: Set /etc/issue to Test on test hosts
      ansible.builtin.copy:
        content: "Test"
        dest: /etc/issue
      when: inventory_hostname in groups.test

    - name: Set /etc/issue to Production on prod hosts
      ansible.builtin.copy:
        content: "Production"
        dest: /etc/issue
      when: inventory_hostname in groups.prod
```

---

## How to run

```bash
cd /home/student/ansible
ansible-navigator run playbooks/issue.yml -m stdout
```

---

## Verification

```bash
ansible dev -a 'cat /etc/issue'
# Expected: Development

ansible test -a 'cat /etc/issue'
# Expected: Test

ansible prod -a 'cat /etc/issue'
# Expected: Production
```

---

## Common exam traps

- **`node5` (balancers) gets all three tasks skipped** — it is not in dev, test, or prod. Its `/etc/issue` is left unchanged. This is correct behaviour.
- **`content:` replaces the entire file** — no need to use `lineinfile` or `replace`. `copy` with `content:` is the simplest and most reliable approach.
- **No trailing newline confusion** — if the exam checks the exact string, a trailing `\n` may cause a mismatch. Test with `cat /etc/issue` and compare exactly.
- **`when: inventory_hostname in groups.dev`** — the `in` operator works on the list returned by `groups.dev`. Do not use `==` which would compare a string to a list.
